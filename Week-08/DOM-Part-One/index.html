The DOM: Part One

Introducing the DOM (Document Object Model

The DOM is simply an object. It is no different than the objects you create in OOP. The DOM represents our entire HTML page. It provides, like objects do, properties and methods that allow us to create, read, update, and delete various pieces of our HTML page.

The DOM is not exclusive to JavaScript, but often the two are used in tandem. The browser's JavaScript API provides a way for you to work with the DOM. On page load, the JavaScript API creates a new version of your API page. It first creates the document object then proceeds to populate it with childrend nodes that represent the other HTML elements you have on the page. Each of these is a different object type, but all inherit from the main object types, Element, Node, and EventTarget.

The DOM Standard is a specification defined by the W3C on what components the browser API must provide to be DOM standard.

Elements are simply HTML blocks. <body> is an element. <div> is an element. <a> is an element. All HTML tags are elements.

Selecting Elements of the DOM with JavaScript

When the DOM is created it creates an object for us called document. document is a representation of our entire HTML page. document is also our entry point when we want to select elements from our page. It has the same properties as any HTMLElement including the selection methods that we're going to look at below.

You can select HTML elements using CSS selectors in JavaScript. The JavaScript document object provides a few ways to do this.

The following are the older antiquated methods that are no longer commonly used, but good to know when working with legacy code. Perform these steps in your main.js file.

  2. Select and store this element into a variable using the method document.getElementById().

If you open your console, you will notice that the document.getElementsByTagName() and the document.getElementsByClassName() returned HTMLCollections. The document.getElementById() returned an HTMLSpanElement.

HTMLElements & HTMLCollections

Remember: ALL HTML tags are elements. However, when we attempt to access them they can be returned in two different ways: HTMLElement or HTMLCollection. So what's the difference?

An HTMLElement represents an HTML element. This object contains many methods and properties that allow us to work with a specific HTML chunk. HTMLElements are returned when we select a unique element. What makes an element unique can vary with being as specific as an ID or as broad as the document order. That is why the older select methods return either a collection or an element. Because IDs are specific and unique, there can only be one element on the page with it. Classes and tag names are not unique, so we are returned a collection.

An HTMLCollection is an array of HTMLElements in order of placement in the document. Once you have the collection you can specifically select an HTML element from it by using an array index. For example, to select the first paragraph tag in this document you would run:

const ps = document.getElementsByTagName('p');
let firstP = ps[0];

The first line returns all the paragraph elements in the document as an HTMLCollection. The second line selects the HTMLElement at the index of 0.
REMEMBER ALL ARRAYS IN JAVASCRIPT ARE 0 (ZERO) INDEXED!

The older methods are anitquated (old & useless) because they're not predictable. If I have one (which you should only ever have one) <h1> element on the page, selecting it by the tag name (.getElementsByTagName()) creates an extra step for me. In order for me to access the element, I have to use an index:

const h1s = document.getElementsByTagName('h1');
let h1 = h1s[0];

This absurdity led to the better newer (well, since IE 8) methods of .querySelector() and .querySelectorAll().

querySelector() & querySelectorAll()

Remember: An HTMLElement represents an HTML element. An HTMLElement is an object created by DOM API that contains many methods and properties including .querySelector() and .querySelectorAll() which allows us to select other HTMLElements within scope the HTMLElement we call the method from.

querySelector('css-selector statement') returns one (and ONLY EVER one) HTML element. Whether you select by tag, class, ID, or attribute, you will always get back only the first instance the selector finds.

  1. Select and store the h2 tag on the page using document.querySelector().
  2. Select and store this element using the ID using document.querySelector().
  3. Select and store this element using the class using document.querySelector(). Hi Boo

querySelectorAll('css-selector statement') returns a collection (and ALWAYS a collection) of HTML elements. When you select by tag, class, ID, or attribute, you will always get back an array of HTML elements, even if there is only one element.

  1. Select and store all the article tags on the page using document.querySelectorAll().
  2. Select and store all the paragraph tags on the page using document.querySelectorAll().
  3. Select and store all of these elements using the class using document.querySelectorAll().

DOM Walking

HTMLElements are objects that represent an HTML element on your HTML page (document). HTMLElement inherits from the Document object (document). This means methods like .querySelector() and .querySelectorAll() are available to it. But that's not the best part! The best part is that they search within scope of the HTMLElement they're called from.

For example, if I wanted to grab all the <li> elements within an <ol> element, I can do the following:

const ol = document.querySelector('ol');
let lis = ol.querySelectorAll('li');

On the first line we select the <ol> element in our document. Remember this will return the first instance of it only because we are using querySelector().
Next we select all the <li>s within that <ol> element scope. That means we're not getting the <li>s from the other <ol> elements; only the one we selected.

The power of this is our ability to use a combination of HTMLElements and .querySelector() to walk our DOM and specifically access the HTMLElements we want. However we are not restricted to walking the DOM this way. We can also use simple HTML walking CSS selectors to do the same thing:

let lis = document.querySelectorAll('ol:first-child li');

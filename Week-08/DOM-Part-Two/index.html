The DOM: Part Two

HTMLELements & HTMLCollections/NodeLists

As stated before, the DOM is simply an object. It is no different than the objects you create in OOP. The DOM represents our entire HTML page. It provides, like objects do, properties and methods that allow us to create, read, update, and delete various pieces of our HTML page.

The HTMLElement is an object representing your HTML tag. The HTMLCollection is a collection of HTMLElements. The NodeList is a collection of every node. This includes all the HTMLElement nodes and Text nodes that may be apart of the collection.

MOST THINGS IN JAVASCRIPT ARE OBJECTS. OBJECTS HAVE PROPERTIES AND METHODS. THIS IS A VERY IMPORTANT CONCEPT YOU NEED TO UNDERSTAND!

Objects

In standard Object Oriented Programming, Classes are created as blueprints of attributes and functionality. The attributes and functionality are better known as Properties and Methods. If we want a subset of those methods an properties we can create a new class that inherits from the class and adjust the visibility using public, private, and protected visibility modifiers. This is the process in Java, PHP, C++, Ruby, and many other languages.

HOWEVER, JavaScript doesn't use classes, it uses object prototypes exclusively. (Yes I know about the ECMAScript CLASS structure, but this still uses prototyping under the hood). We can create our own objects easily:

  2. Give you object a name, age, gender, and an array of interests.
  3. Give your object a function called greeting and have it output a welcome message.
  4. Give your object a function called bio and have it output their name, age, gender and interests in a logical sentence.
  5. Call the greeting().
  6. Call the bio().
  7. Output the name and age.

HTMLElement Properties and Methods

An HTMLElement is an object that represents your HTML tag. In HTML a tag describes its content. For example, <article> tells the browser that the content contained inside is semantically connected to an article. JavaScript doesn't care about semantics, but it does care about the tag name. Every HTML tag will become an HTMLElement in the DOM. In addition, the HTMLElement inherits properties and methods from various other objects available: Element, Node, and EventTarget objects. This means we have many properties and methods available to us to work with our HTMLElement.

  1. Select and store the <h1> element.
  2. Change the text content of the h1 element object.
  3. Change the text colour of the h1 element object.
  4. Make the h1 object text uppercase

The last step's syntax is an interesting demonstration of prototypal inheritance:

h1.textContent = h1.textContent.toUpperCase();

h1 is an object. textContent is a property of h1. textContent's value is a string (which is actually a text node object). The text node object inherits from String. That is why we have access to the String method .toUpperCase(). This whole line contains a lot of objects giving us a lot of capability.

This leads to another important concept in JavaScript: A variable/property value can be of any type! That includes Numbers, Strings, Booleans, Arrays, Objects, and Functions. That means a property of an object can have the value of an entire other object, and in regards to the DOM, often does!

HTMLCollection/NodeList Properties and Methods

A NodeList is an object. It acts much like an Array object as it contains a zero-index list of node elements. In this case, the list is entirely made up of HTMLElements and text nodes.

When you select multiple HTMLElements using .querySelectorAll() you will be returned a NodeList object. In addition, an individual HTMLElement will contain a property called childNodes that has a NodeList object as a value. If it isn't obvious, this NodeList contains all the children HTML elements under the selected element.

  1. Select and store the first <ol> element in the HTML document.
  2. Select an store the property's value that contains a NodeList.
  3. Select an store the property's value that contains an HTMLCollection.
  4. What is the difference between the two collections values?

HTMLCollections and NodeList behave much like arrays, however there are some differences between each. For example, iterating through an HTMLCollection can be achieved with a for loopand a for/of loop, or by using a while/do while structure or recursion. It does not have access to the method .forEach() though. NodeLists can use any of the iterator methods including the .forEach() method. This is another reason that .querySelectorAll() is so cool as it ALWAYS returns a NodeList.

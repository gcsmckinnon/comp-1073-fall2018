Objects: Part One

Object Recap

Objects, like in most languages, are a way for us to encapsulate code under one idea. Generally an object is best described using a noun (a  person, place, or thing).

Properties are basically descriptive words (adjectives) that can be used to define properties or attributes about an object.Below, we see an object defined with 3 properties:name,  age, and interests. 

let person = {
  name: "Shaun",
  age: 39,
  interests: ["programming", "Diablo", "cage fighting"]
}

Methods are actions an object can perform. For example an object of type Shaun will likely be able to talk and walk. 

let person = {
  name: "Shaun",
  age: 39,
  interests: ["programming", "Diablo", "cage fighting"]
  talk: function () {
    console.log("I'm 39");
  },
  walk: function () {
    posX += 5;
  }
}

JavaScript Object Properties

Objects are an integral part to JavaScript. In JavaScript, ALMOST everything is an object. That's very cool if you consider that objects give you properties and methods. It's like having extra functionality on every variable, function, string, number, and array (for example) that you place in your code!

Let's look at what properties a variable with a string has: 

let name = "Shaun McKinnon";
let nameProperties = Object.getOwnPropertyNames(name);
console.log(nameProperties.join(', '));

So we returned a block of numbers and "length". The block of numbers represents the character indexes in our string. The length property is the total character count of our string. If we wanted to access a character using one of the index properties, we can simply do this using one of the numbers. 

console.log(name[1]); // "h"
console.log(name[11]); // "n"
console.log(name["length"]); // 1

Pretty cool eh? But we're not done. Let's look at the properties from our person object:

  1. Create the following object:
    let person = {
  name: "Shaun",
  age: 39,
  interests: ["programming", "Diablo", "cage fighting"],
  talk: function () {
    console.log("I'm 39");
  },
  walk: function () {
    posX += 5;
  }
}
  2. Output all the properties on the object
    let objProps = Object.getOwnPropertyNames(person);
console.log(objProps.join(', '));
// name, age, interests, talk, walk

Hold the phone:
TALK AND WALK AREN'T PROPERTIES!!! They're methods!
But they are! Properties are simply named values. No different than a variable. The value can be any data type: string, number, array, function, or even another object.

Accessing Object Properties

Externally:
We have 2 options for accessing an object property. We can using the .dot syntax: 

console.log(person.name);

Or, we can using the ["index"] syntax: 

console.log(person["name"]);

Make a note though that .dot syntax is without any quotes and ["index"] syntax is. The cool thing about ["index"] syntax is you now you have the ability to dynamically access object properties if you needed to.
Let's populate the following structure:

Name:
Age:
Interests:
Bio:
  1. Set the #personName, #personAge, and #personInterests text content to our the corresponding object properties
    const personName = document.querySelector('#personName');
personName.textContent = person.name;

const personAge = document.querySelector('#personAge');
personAge.textContent = person.age;

const personInterests = document.querySelector('#personInterests');
personInterests.textContent = person.interests.join(', ');

Accessing the functions is done the same way as accessing any property. The difference lies when you want to execute() them. You use the (parenthesis) to tell the JavaScript interpreter to call the function .

  1. Create a new function definition for the property
    bio in the person object 
    bio: function () {
  return `Hello I am Shaun and I'm a 39 year old computer programmer!`;
}
  2. Set the #personBio text content to return value of the bio property function
    const personBio = document.querySelector('#personBio');
personBio.textContent = person.bio();

Internally:
Accessing properties within the object (internally) involves using the very important keyword this. In objects this represents the owner object which in this case is person.

We can see this by console logging this but we will need to do it within the object. Let's add it to our bio method as we're already calling it: 

bio: function () {
  console.log(this);
  return `Hello I am Shaun and I'm a 39 year old computer programmer!`;
}
// {name: "Shaun", age: 39, interests: Array(3), talk: f, walk: f, bio: f}

As you can see, this IS our person object. You can see all the properties, and, if you're outputting to the console, you can click the arrows to div into what each property is.

So, if we wanted to DRY (Don't Repeat Yourself) our bio function definition, we could simply use this to reference our properties:

  1. Using the this keyword, replace "Shaun", 39, and "computer programmer" with the correct object property values 
    bio: function () {
  console.log(this);
  return `Hello I am ${this.name} and I'm a ${this.age} year old ${this.interests[0]}`;
}

Because properties can have any value type they can hold function defintions. We saw the power of this before with simple values. What if our values are functions? Can we still use this?
Yes! Yes we can!

  1. First, modify the bio method so it says "Hello, I am ${this.name} and I'm a ${this.age} year old interested in ${this.interests[0]}." 
    bio: function () {
  console.log(this);
  return `Hello I am ${this.name} and I'm a ${this.age} year old interested in ${this.interests[0]}`;
}
  2. Next, add a toHTML method that creates a new dif and adds it to page based on a selector you pass it 
    toHTML: function (selector) {
  let div = document.createElement('div');
  div.textContent = this.bio();
  document.querySelector(selector).append(div);
}
  3. Last, call the new toHTML method externally passing '#toHTMLBio' as the argument 
    person.toHTML('#toHTMLBio');

Creating New Object Properties

Up to this point, we've been creating object properties by simply adding to our person object structure. It now currently resembles this: 

let person = {
  name: "Shaun",
  age: 39,
  interests: ["programming", "Diablo", "cage fighting"],
  talk: function () {
    console.log("I'm 39");
  },
  walk: function (s) {
    posX += 5;
  },
  toHTML: function (selector) {
    let div = document.createElement('div');
    div.textContent = this.bio();
    document.querySelector(selector).append(div);
  },
  bio: function () {
    console.log(this);
    return `Hello I am ${this.name} and I'm a ${this.age} year old interested in ${this.interests[0]}`;
  }
}

This is a lot to maintain, and, to be honest, not practical. Sometimes you may want to create a new property based on a condition. Or a bunch of new properties using a loop. Because JavaScript is a loosey goosey, duck-typed (if it walks like a duck and quacks like a duck it must be ...), we have the ability to add a new property whenever we want.

  1. Add a new property called hasPets and set it's value to true (if you have pets) or false (if you don't) 
    person.hasPets = true;
  2. Add a new property called petCount and set it's value to the number of pets you have (0 if you have none) 
    person.petCount = 3;
  3. Add a new property called petToHTML and set it's value to the following function defintion 
    person.petToHTML = function (selector) {
  let div = document.createElement('div');

  if (this.hasPets) {
    div.textContent = `${this.name} has ${this.petCount} pets.`;
  } else {
    div.textContent = `${this.name} doesn't have any pets.`;
  }

  document.querySelector(selector).append(div);
}
  4. Lastly, call the new petToHTML method passing '#doYouGotPets' as the argument 
    person.petToHTML('#doYouGotPets');
